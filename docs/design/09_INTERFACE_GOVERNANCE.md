# 09 - Interface Governance

## Purpose

Design governance protects AS6 from visual drift.

## Master Rule

Screen 3 Living Space is the master visual reference for the whole product.

## Acceptance Checklist

Every new state must answer yes:

- Does it feel like AS6?
- Does it preserve the master atmosphere?
- Does it keep the same living space logic?
- Does it avoid unrelated page design?
- Does it keep text readable?
- Does it make the next step clear?

## Failure Condition

If a state looks like another product, it fails governance.

## Approval Rule

A new state is approved only when it feels like the same living space changing attention.

# AS6 Living Space Design System v2

Статус: обязательный визуальный стандарт

## 1. Характер

Интерфейс должен ощущаться:

- спокойным;
- светлым;
- умным;
- человечным;
- живым;
- точным;
- премиальным без демонстративной роскоши.

## 2. Цветовая система

### Основной текст

- `--as6-ink-strong: #245a82`
- `--as6-ink: #3d7094`
- `--as6-ink-soft: #7299b5`
- `--as6-ink-faint: rgba(61, 112, 148, 0.48)`

### Пространство

- `--as6-sky-top: #eef9ff`
- `--as6-sky-middle: #dceffc`
- `--as6-sky-bottom: #f5fbff`
- `--as6-light-warm: #fff9dd`
- `--as6-surface: rgba(255, 255, 255, 0.30)`
- `--as6-surface-strong: rgba(255, 255, 255, 0.58)`

### Границы и свечение

- `--as6-border: rgba(255, 255, 255, 0.82)`
- `--as6-border-focus: rgba(74, 129, 166, 0.42)`
- `--as6-glow: rgba(255, 255, 255, 0.72)`
- `--as6-shadow: rgba(64, 119, 157, 0.12)`

### Состояния

- success: спокойный зелёно-голубой;
- warning: тёплый янтарный без агрессивного оранжевого;
- error: приглушённый красный;
- information: основной синий AS6.

## 3. Типографика

Основной стек:

`Inter, "Segoe UI", Arial, sans-serif`

### Уровни

- Display: 64–96 px, weight 300;
- H1: 48–72 px, weight 300;
- H2: 32–44 px, weight 400;
- H3: 22–28 px, weight 400;
- Body Large: 18 px;
- Body: 15–16 px;
- Caption: 11–13 px;
- Eyebrow: 10–11 px, uppercase, tracking 0.14–0.18em.

Большие заголовки используют отрицательный letter-spacing и короткие строки.

## 4. Пространственная сетка

- максимальная ширина: 1440 px;
- рабочая ширина: 1280–1320 px;
- минимальный внешний отступ desktop: 32 px;
- mobile: 16 px;
- базовый шаг: 8 px;
- крупные интервалы: 48, 64, 80, 96 px.

## 5. Радиусы

- small: 12 px;
- medium: 18 px;
- panel: 28–34 px;
- pill: 999 px;
- круговые смысловые узлы: 50%.

## 6. Поверхности

Разрешены:

- прозрачные светлые панели;
- мягкий blur;
- внутреннее свечение;
- тонкая белая граница;
- едва заметная тень.

Запрещены:

- тяжёлые серые карточки;
- чёрные панели;
- неоновые градиенты;
- толстые рамки;
- чрезмерные drop-shadow;
- визуальная имитация криптопродукта.

## 7. Компоненты

Обязательная библиотека:

- LivingShell;

- LivingConfirmation;
- LivingToast;
- LivingDialog.

## 8. Анимации

Анимация объясняет изменение состояния.

Разрешены:

- fade;
- мягкое перемещение 4–16 px;
- изменение прозрачности;
- плавное расширение пространства;
- медленное дыхание световых узлов.

Обычно: 180–320 ms.  
Атмосферные движения: 6–18 секунд.

Обязательна поддержка `prefers-reduced-motion`.

## 9. Интерактивность

Каждый интерактивный элемент должен иметь:

- hover;
- focus-visible;
- active;
- disabled;
- loading;
- success;
- error.

Фокус клавиатуры не удаляется.

## 10. Mobile

Mobile — не уменьшенная desktop-версия.

На mobile:

- сохраняется один смысловой центр;
- вторичные панели переходят в последовательный поток;
- Universal Command остаётся доступным;
- нижняя навигация допускается только для главных состояний;
- минимальная зона касания — 44×44 px.

## 11. Контроль качества

Новый компонент не принимается, если:

- не наследует токены;
- использует произвольный цвет;
- создаёт отдельный визуальный язык;
- не имеет keyboard focus;
- нарушает reduced motion;
- выглядит как отдельная SaaS-страница;
- не связан с состоянием Living Space.

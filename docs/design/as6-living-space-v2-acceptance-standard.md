# AS6 Living Space v2 — Acceptance Standard

## 1. Кликабельный prototype

Prototype считается готовым, когда доступны:

- Living Home;
- AI Conductor;
- Relations;
- Projects;
- Documents;
- Knowledge;
- Living Blog;
- Universal Search;
- Universal Command;
- Settings.

## 2. Обязательные сценарии

### Сценарий A — первый вход

1. Пользователь входит.
2. Открывается Living Home.
3. Интерфейс объясняет текущее состояние.
4. Видно одно главное действие.
5. Не требуется обучение.

### Сценарий B — работа с AI

1. Пользователь формулирует намерение.
2. AI показывает понимание задачи.
3. AI показывает план.
4. Опасные изменения требуют подтверждения.
5. После выполнения показывается результат и история.

### Сценарий C — поиск

1. Открытие через `Ctrl + K`.
2. Поиск по нескольким типам сущностей.
3. Переход к результату без смены оболочки.
4. Поддержка клавиатуры.

### Сценарий D — CRM

1. Пользователь видит клиентов, требующих внимания.
2. Видит причину.
3. Видит следующее действие.
4. Может поручить действие AI.

## 3. Visual acceptance

- соответствует атмосфере Master Screen;
- нет тёмного legacy UI;
- нет произвольных цветов;
- нет тяжёлой административной панели;
- один смысловой центр;
- одна главная кнопка;
- понятность без инструкции;
- корректная desktop и mobile композиция.

## 4. Technical acceptance

- один LivingShell;
- состояния не дублируют оболочку;
- маршруты lazy-load при необходимости;
- keyboard navigation;
- reduced motion;
- semantic HTML;
- отсутствие console errors;
- production build проходит;
- legacy scan проходит;
- Lighthouse не является единственным критерием качества.

## 5. Accessibility acceptance

- контраст текста достаточен;
- интерактивные элементы доступны клавиатурой;
- focus-visible заметен;
- формы имеют label;
- ошибки связаны с соответствующими полями;
- motion можно отключить;
- screen reader получает понятные названия областей.

## 6. Запрет реализации

Нельзя переносить новый модуль в production до утверждения его состояния в
кликабельном prototype.

Исключение: security, data integrity или production incident repair.

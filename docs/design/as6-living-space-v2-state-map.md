# AS6 Living Space v2 — State Map

## Корневой маршрут

`/app`

Корневой маршрут всегда открывает единый Living Shell.

## Состояния пространства

| Состояние | Внутренний идентификатор | Назначение |
|---|---|---|
| Дом | `home` | Главный фокус и текущее состояние бизнеса |
| AI-дирижёр | `conductor` | Намерение, план, выполнение и результат |
| Клиенты | `relations` | Отношения, риски и следующие действия |
| Проекты | `projects` | Результаты, этапы и препятствия |
| Документы | `documents` | Рабочие документы в контексте |
| Знания | `knowledge` | Связанные знания и история решений |
| Блог | `blog` | Интерактивные материалы |
| Поиск | `search` | Универсальный поиск |
| Команды | `command` | Universal Command |
| Настройки | `settings` | Параметры пространства |

## Правило URL

URL отражает состояние, но не создаёт отдельное приложение.

Допустимые маршруты:

- `/app`
- `/app/conductor`
- `/app/relations`
- `/app/projects`
- `/app/documents`
- `/app/knowledge`
- `/app/blog`
- `/app/settings`

Все маршруты используют один `LivingShell`.

## Основной пользовательский поток

1. Регистрация или вход.
2. Открытие `/app`.
3. Living Home показывает текущий фокус.
4. Пользователь формулирует намерение.
5. AI Conductor собирает контекст.
6. Показывается понятный план.
7. Пользователь подтверждает изменения.
8. AS6 выполняет действие.
9. Пространство обновляется без резкой смены интерфейса.
10. Результат регистрируется в истории.

## Универсальный поток ошибки

1. Обнаружить проблему.
2. Объяснить простыми словами.
3. Сохранить введённые данные.
4. Предложить безопасное следующее действие.
5. Дать повторить операцию.
6. Зарегистрировать diagnostic event.

## Переход legacy

Старые маршруты временно перенаправляются в `/app`.

После переноса всех функций:

1. проверить отсутствие входящих ссылок;
2. проверить отсутствие импортов;
3. удалить legacy route aliases;
4. удалить legacy components;
5. повторить production validation.

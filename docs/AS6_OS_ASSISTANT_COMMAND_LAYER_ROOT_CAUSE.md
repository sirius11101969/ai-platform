# AS6 OS Assistant Command Layer Root Cause

- Root cause: AS6 visual brand is stronger after OS Brand System Refinement, but the user still needs a persistent command/assistant affordance visible across CRM work.
- Interface gap: CRM has AS6 Assistant blocks, but no always-visible lightweight command layer.
- Risk: AS6 may still feel like dashboard navigation instead of active AI Operating System.
- Resolution: add non-destructive CSS command layer on /crm with visible AS6 command entry, next action prompt and keyboard hint.
- Page changed: /crm.
- UI location changed: bottom-right floating layer and CRM Workspace top surface.

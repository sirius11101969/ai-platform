# AS6 Universe Workflow

## Version

AS6_UNIVERSE_WORKFLOW_V1

## Purpose

This workflow defines how every AS6 Living Space is created, reviewed and promoted to Golden Master.

## Workflow

Every Living Space must follow this sequence:

1. Space Passport
2. Visual DNA
3. OpenAI image prompt
4. Generated image
5. Review against Golden Master 01 Focus
6. Corrections
7. Golden Master checks
8. Registry update
9. Freeze if approved

## Required Checks

A Living Space cannot become Golden Master unless all checks pass:

| Check | Requirement |
|---|---|
| Recognizable without text | User can identify the space without reading title or labels. |
| AS6 identity preserved | User immediately feels this is AS6. |
| Visual distance | Space is not confused with any other AS6 Living Space. |
| Constant atmosphere | Background, typography and structure match AS6 standards. |
| One main visual center | No competing central objects. |
| Emotional match | Space creates its intended emotion. |
| Practical clarity | User understands the next useful action. |

## Constant Inheritance

Every space inherits from GM-01 Focus:

- base background;
- typography;
- top area logic;
- right panel logic;
- bottom card logic;
- input field logic;
- glass language;
- calm premium tone.

## Unique Identity

Every space must change:

- central symbol;
- geometry;
- material;
- object language;
- connection language;
- accent light;
- emotional signature.

# AS6 Calm Atmospheric Interface Standard

## Статус

APPROVED / CANONICAL

## Назначение

Стандарт фиксирует ощущение утверждённого CRM Golden Master и определяет, какие свойства должны быть перенесены на пространства «Фокус» и «Финансы».

## Базовый принцип

**AS6 не показывает интерфейс. AS6 создаёт спокойное пространство, в котором информация открывается естественно.**

## 1. Calm Attention

Интерфейс не просит внимания и не конкурирует с пользователем.

- Второстепенный текст тихий.
- Заголовки читаемы, но не агрессивны.
- Центральная сцена остаётся главным объектом.
- Одновременно не может быть много сильных акцентов.

Тест: если взгляд прыгает между заголовками и панелями, иерархия нарушена.

## 2. Invisible Interface

Пользователь должен замечать содержание раньше контейнера.

- Рамка не используется, если смысл читается без неё.
- Панель не используется, если информация сохраняет структуру без неё.
- Стекло не применяется ради декоративного эффекта.

## 3. Progressive Visibility

Служебный интерфейс проявляется только при необходимости.

- Поле общения почти исчезает в состоянии покоя.
- При фокусе или наведении его граница может мягко проявляться.
- Дополнительные легенды и аналитические пояснения открываются по запросу.

## 4. Interface Dissolution

Навигация, рекомендации и элементы взаимодействия воспринимаются как часть атмосферы.

Запрещены яркие постоянные рамки вокруг:

- названия пространства;
- рекомендации;
- строки общения;
- неинтерактивных информационных блоков.

## 5. Atmospheric Interface

Все элементы находятся в одной непрерывной среде.

- Нет ощущения набора окон.
- Нет тяжёлых карточек.
- Нет резких границ между функциональными зонами.
- Воздух создаётся отсутствием лишних контейнеров, а не пустотой ради пустоты.

## 6. Semantic Light

Свет означает, что система живёт, действует или обнаруживает изменение.

Светятся:

- процессы;
- связи;
- узлы;
- активные показатели;
- важные текущие изменения.

Матовым текстом объясняются:

- названия;
- смысл;
- навигация;
- аналитические выводы;
- рекомендации.

## 7. Calm Typography

Основной текст — матовый серо-синий.

Требования:

- достаточная читаемость на светлом фоне;
- отсутствие собственного свечения;
- отсутствие насыщенного синего и чистого чёрного;
- отсутствие одинакового сильного веса у всех заголовков;
- второстепенная информация заметно спокойнее основной.

Критерий: пользователь должен почувствовать улучшение читаемости, но не заметить, что типографика стала тяжелее.

## 8. Visual Priority

Базовая последовательность восприятия:

1. Центральный живой процесс.
2. Текущее состояние пространства.
3. Ключевые показатели.
4. Влияющие объекты.
5. Правая аналитика.
6. Рекомендация.
7. Навигация и ввод.
8. Контейнеры и рамки.

Пункты 7–8 не должны визуально конкурировать с пунктами 1–4.

## 9. Space-Specific Identity

При переносе стандарта между пространствами сохраняются:

- спокойствие;
- воздух;
- типографическая иерархия;
- растворённый интерфейс;
- семантика света;
- отсутствие конкурирующих контейнеров.

Не переносятся:

- уникальная геометрия;
- закон пространства;
- цветовой акцент;
- живые объекты;
- характер связей.

## 10. Acceptance Tests

Экран соответствует стандарту, если:

- после 10 секунд просмотра нет ощущения визуальной усталости;
- центральный процесс определяется без поиска глазами;
- второстепенные подписи доступны, но не требуют внимания;
- рамки не воспринимаются как самостоятельные объекты;
- текст не перекрывает и не заглушает визуальный процесс;
- экран ощущается как единая среда;
- пользователь видит спокойствие раньше функциональности.

## Наследование

Этот стандарт обязателен для следующих циклов:

- `GM-01 Focus Calm Alignment`
- `GM-02 Finance Calm Alignment`
- всех последующих AS6 Living Spaces.

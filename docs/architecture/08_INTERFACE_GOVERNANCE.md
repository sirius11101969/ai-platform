# 08 - Interface Governance

## Purpose

Interface Governance ensures every AS6 interface remains a state of one Living Space instead of becoming an isolated screen.

## Mandatory Rules

- One visual language.
- One atmosphere.
- One navigation model.
- One interaction philosophy.
- One emotional experience.

## Never Do

- Do not redesign each page independently.
- Do not introduce a different visual identity for another module.
- Do not increase visual complexity to express intelligence.

## Always Do

- Keep the light blue calm atmosphere.
- Keep continuity between states.
- Explain recommendations in human language.
- Preserve whitespace and readability.
- Let behavior communicate intelligence.

## State Checklist

Every new state must preserve:

- Living Space composition
- Calm Growth principles
- AS6 Design Language
- Interface Standard
- Emotional continuity

## Acceptance Rule

If a screenshot of a new state looks like another product, the design fails governance.

If the screenshot is immediately recognizable as AS6 without seeing the logo, governance passes.

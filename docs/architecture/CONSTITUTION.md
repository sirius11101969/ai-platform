# AS6 Constitution

## Purpose

This document connects the core AS6 architecture standards into one product constitution.

Any meaningful change in AS6 must respect these standards.

## Core Standards

1. Diagnostics First Standard
2. Product Knowledge Architecture
3. Knowledge Lifecycle Architecture
4. Living Space Architecture
5. Calm Growth Design System
6. AS6 Design Language
7. Interface Standard
8. AS6 Architecture Manifesto

## Constitution Rule

AS6 is not developed as a set of disconnected features.

AS6 is developed as one coherent business operating environment.

Every change must improve at least one of these dimensions:

- clarity
- calmness
- continuity
- diagnostic confidence
- knowledge quality
- user understanding
- business usefulness

## Diagnostics First

Every change starts with diagnosis and ends with evidence.

## Knowledge First

Every important decision must become structured product knowledge.

## Lifecycle First

Knowledge must move from idea to operation and feedback.

## Living Space First

User-facing experience must behave as one living space with many states.

The interface should evolve instead of jumping between unrelated screens.

## Calm Growth First

More intelligence must create more calm, not more complexity.

## Design Language First

Every visible state must feel like AS6:

- light blue atmosphere
- calm space
- clear typography
- living relationships
- soft glass surfaces
- human language
- no heavy dashboard feeling

## Final Principle

AS6 must help the user feel that business becomes simpler, clearer and stronger.

The system should not demand attention.

The system should calmly prepare the next useful step.

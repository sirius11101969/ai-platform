MVP delivery sequence

  • Data model: workspaces, users, leads, pipeline, notes, appointments, proposals.
  • Backend API + CRM dashboard + landing lead forms.
  • Telegram qualification + n8n lead routing and reminders.
  • Basic AI secretary, proposal generation and booking flow.
  • Sentry/logs/health checks before first paid pilots.

Growth migration gates

  • Add queues when jobs slow API or require reliable retry.
  • Separate agents when AI execution needs independent scale or monitoring.
  • Add voice only after CRM + booking loop is stable.
  • Add Grafana/Prometheus when queue/provider failures need operational visibility.
  • Keep microservices out until team/load creates a real need.

Scale readiness gates

  • Extract services only by load, ownership or release cadence.
  • Introduce event bus when multiple consumers need durable domain events.
  • Add vector/memory stores only when RAG/context measurably improves agent performance.
  • Use read replicas when reporting impacts transactional workload.
  • Adopt advanced tracing after service boundaries exist.

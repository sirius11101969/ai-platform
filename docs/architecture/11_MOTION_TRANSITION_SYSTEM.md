# AS6 Motion & Transition System v1.0

## Purpose

Motion & Transition System defines how AS6 Living Space becomes alive.

Motion is not decoration. Motion explains continuity, state change and user feedback.

## Main Rule

AS6 never switches between pages.

AS6 transforms one Living Space state into another.

## Motion Principles

### Calm Motion

All movement is smooth, slow enough to understand and fast enough to feel responsive.

### Meaningful Motion

Every animation must answer at least one question:

- what changed;
- why it changed;
- where the object moved;
- what state is now active.

### Spatial Continuity

Motion preserves the same environment:

- atmosphere;
- light;
- depth;
- material;
- scale;
- panel behavior;
- brand recognition.

### One Primary Transformation

During state change, the central geometry is the main moving object.

Panels and tools follow it with lower priority.

## Transition Types

### Living Space to Focus

The main sphere compresses into one stable center.

Emotion: calm concentration.

### Focus to Thinking

The single center releases asymmetric orbital fragments.

Emotion: idea formation.

### Thinking to Analytics

Orbits crystallize into clear data facets.

Emotion: clarity.

### Analytics to Decision

Crystal facets align into converging vectors.

Emotion: confidence.

### Decision to Strategy

The selected vector extends into a long path with checkpoints.

Emotion: direction.

### Strategy to Planning

The path bends into time arcs around the priority center.

Emotion: predictability.

### Planning to Automation

Time arcs become continuous process streams.

Emotion: ease.

### Automation to Living Ecosystem

Streams connect into a living node network.

Emotion: growth.

### Living Ecosystem to Evolution

Network motion unfolds into an expanding spiral.

Emotion: future growth.

### Any State to Diagnostics

The current geometry slows down, becomes transparent and receives scanning layers.

Emotion: inspection and control.

### Any State to Knowledge

The current geometry breaks into meaningful connected points and forms a constellation.

Emotion: context and discovery.

## Panel Motion

Panels never jump.

They:

- fade through depth;
- slide softly within the same spatial plane;
- reduce priority during central transformation;
- reappear only after the new state geometry is readable.

## Light Motion

Light changes before the user consciously notices the state change.

Rules:

- no flashes;
- no aggressive contrast;
- no decorative glow;
- light guides attention to the next central geometry.

## Microinteractions

Microinteractions must be subtle and useful:

- confirm action;
- show progress;
- reveal relation;
- reduce uncertainty.

They must not compete with the central geometry.

## Timing Rules

- Micro feedback: immediate.
- Panel response: short and calm.
- State transformation: smooth and readable.
- Full environmental transition: continuous and non-disruptive.

## Failure Classes

- AS6_PAGE_SWITCH_MOTION_DRIFT: transition feels like page replacement.
- AS6_DECORATIVE_MOTION_NOISE: animation does not explain meaning.
- AS6_GEOMETRY_TRANSITION_DUPLICATION: transformation repeats another state.
- AS6_LIGHT_SHOCK_DRIFT: light change is too sharp.
- AS6_PANEL_JUMP_DRIFT: panels appear as separate windows.

## Approval Gate

A motion pattern is approved only when:

- it preserves AS6 identity;
- it explains the change;
- it keeps user calm;
- it keeps one central transformation;
- it does not create interface noise;
- it supports Living Space continuity.

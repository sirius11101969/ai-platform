# AS6 Visual Grammar v1.0

## Purpose

AS6 Visual Grammar defines the formal visual language of AS6 Living Space.

It converts the approved design architecture into a repeatable grammar that can be used by designers, engineers and AI agents.

AS6 is not a collection of screens.

AS6 is a Living Space made from repeatable visual primitives, emotional meanings and spatial composition rules.

## Core Formula

AS6 visual language is built as:

```text
Light
  -> Form
  -> Geometry
  -> Composition
  -> Space
  -> State
  -> Living Space
```

A new interface state is valid only when it can be expressed through this grammar.

## Grammar Operators

AS6 states are composed from primitives with operators.

### Center

Places one primitive as the primary attention object.

Example: `Center(Core) = Focus`.

### Orbit

Creates movement around a central meaning.

Example: `Sphere + Orbit = Thinking`.

### Connect

Links nodes into a meaningful system.

Example: `Node + Connect + Flow = Living Ecosystem`.

### Converge

Brings multiple directions into one decision.

Example: `Ray + Converge + Core = Decision`.

### Crystallize

Turns scattered information into clear structure.

Example: `Node + Grid + Crystal = Analytics`.

### Scan

Adds transparent diagnostic layers over an existing state.

Example: `Any State + Layer + Pulse = Diagnostics`.

### Expand

Opens a stable form into future growth.

Example: `Core + Spiral + Flow = Evolution`.

### Sequence

Arranges elements along time or priority.

Example: `Vector + Arc + Checkpoint = Planning`.

## Composition Rules

1. One primary center per state.
2. No more than three attention levels.
3. Secondary elements must support the central geometry.
4. Empty space is an active composition element.
5. Brand atmosphere is always inherited from Master Screen 3.
6. Central geometry must be unique per state.
7. No page-specific visual language is allowed.
8. Motion must express transformation, not replacement.

## State Creation Pipeline

Every new AS6 state must follow this pipeline:

```text
Task
  -> Emotion
  -> Visual Primitive Set
  -> Grammar Formula
  -> Central Geometry
  -> Spatial Composition
  -> Motion Behavior
  -> Living Space Validation
```

## Validation Gate

A new state is accepted only when:

- it has a clear task;
- it has one emotional target;
- it uses allowed AS6 visual primitives;
- it has a written geometry formula;
- it does not duplicate another state;
- it inherits Master Screen 3;
- it can transform smoothly from at least one existing state;
- it is recognizable as AS6 in less than one second.

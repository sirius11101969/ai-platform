# AS6 Governance Controller Convergence Policy

Purpose:

Make autonomous governance diagnostics simpler, faster, deterministic and aligned with current AS6 registries.

Rules:

- Diagnostic flow must validate current AS6 artifacts, not stale legacy assumptions.
- Autonomous knowledge base, incident governance, incident lifecycle and change approval controllers must use one shared convergence contract.
- Unified autonomy orchestrator must aggregate normalized controller results and must never exceed 100 percent.
- Controller diagnostics should be fast and should not run the entire diagnose-all recursively.
- WARN debt may remain visible, but current governance convergence must not fail on historical-only artifacts.

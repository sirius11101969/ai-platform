## Экран 1: живое пространство

Главный экран AS6 строится вокруг текущей бизнес-цели, а не статичного сценария. Цель, действие AS6, показатель результата, подготовленные материалы, цепочка пространств и боковые пояснения ведут в соответствующий рабочий контекст.

- верхняя строка открывает реальные события за текущий день;
- центральная цель выбирается из актуальных приоритетов;
- голосовая кнопка использует распознавание речи браузера;
- светлая и тёмная темы сохраняются между сеансами;
- компания является отдельным рабочим пространством и выбирается в меню под логотипом;
- тарифы ограничивают число собственных пространств: Free/Starter — 1, Pro — 3, Business — 10, Enterprise — до 1000;
- `/` всегда показывает публичный сайт, в том числе после авторизации; рабочее пространство доступно по `/app`.
- HTML не кэшируется между релизами, а хешированные статические ресурсы кэшируются как immutable.

Проверка контракта: `AS6_ROOT="$PWD" ops/bin/as6-control-screen1-interaction-multi-workspace-v1`.

## Тарифы и защищённая оплата

Публичная страница `/pricing` получает цены и лимиты из единого серверного каталога. Для покупки пользователь входит в аккаунт, выбирает своё активное рабочее пространство и переходит на платёжную страницу ЮKassa.

- сумму, валюту и допустимый переход между тарифами определяет только сервер;
- оплатить повышение может только владелец пространства;
- понижение и повторная покупка текущего тарифа блокируются;
- публичный webhook ЮKassa перепроверяет статус, сумму и валюту платежа через API провайдера;
- после подтверждения тариф обновляется у владельца и во всех принадлежащих ему пространствах, поэтому новые лимиты действуют автоматически;
- прямое изменение тарифа или кредитного баланса через настройки запрещено;
- страница возврата из ЮKassa показывает успех только после статуса `paid` в AS6;
- текущий запуск использует разовую активацию без автоматического продления.

Проверка контракта: `AS6_ROOT="$PWD" ops/bin/as6-control-pricing-payment-v1`.
